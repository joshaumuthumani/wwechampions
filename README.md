# WWE Champions Dashboard

A single-repository dashboard for current WWE champions. The frontend reads a local cache at `public/champions.json`; the cache is generated by `scrape.py`, which aggregates WWE.com champion data with Cagematch title metadata.

## Tech Stack

- Frontend: Vite, React, TypeScript, Tailwind CSS
- Scraper: Python, Playwright, BeautifulSoup
- Data cache: `public/champions.json`

## Prerequisites

- Node.js 20+
- Python 3.11+
- Chromium installed through Playwright

## Setup

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python -m playwright install chromium
npm install
```

## Generate Champion Data

Run the scraper manually whenever you want to refresh the cache:

```bash
source .venv/bin/activate
python scrape.py
```

The scraper writes the final array to `public/champions.json`. If Cagematch is unavailable or blocks a request, the script still preserves WWE.com champion records with display-safe fallback values.

## Run the Dashboard

```bash
npm run dev
```

Then open the local URL printed by Vite, usually `http://localhost:5173`.

## Build

```bash
npm run build
```

## Data Shape

```json
{
  "titleName": "United States Champion",
  "championName": "Trick Williams",
  "imageUrl": "https://...",
  "championshipDate": "April 19, 2026",
  "daysAsChampion": 28,
  "lastDefenseDate": "No Title Defenses Yet",
  "daysSinceLastDefense": null,
  "source": {
    "wweUrl": "https://www.wwe.com/...",
    "cagematchUrl": "https://www.cagematch.net/..."
  }
}
```

## Scraper Notes

- WWE.com is parsed from its public superstar/champion listing and champion profile pages.
- Cagematch is loaded with Playwright because it commonly serves a JavaScript gate to non-browser clients.
- Names and titles are normalized for accents, punctuation, curly apostrophes, ampersands, casing, and whitespace.
- The scraper matches exact normalized titles first, then falls back to fuzzy title and champion similarity.
- Missing title defenses are represented as `No Title Defenses Yet` in the JSON and UI.
